在eclipse中单独导入安卓源码中的APK或jar库源码，由于安卓源码工程很大，整个导入安卓源码到eclispe必然会导致ide的卡顿，并且全部导入有一个致命的缺陷，就是APK源码的java部分和资源部分是分开的，不是显示在一块，这样非常不符合习惯，并且频繁的在java和res之间切换很浪费时间。因此这里给出在eclipse单独导入安卓源码的APK或jar类库的方法。

方法

导入APK源码

点击File - Import - Android - Existing Android Code in Workspace，
点击Next，
Select root directory选择APK源码所在目录，在下面的列表中修改项目名称。
取消Copy prjects into workspace的√
点Finish。
这样就导入源码了。

注意事项

有一个问题要非常注意。
由于eclipse导入APK项目之后，会在APK项目目录下生成一个gen目录，这里会有R.java的生成，这个时候需要注意看一下项目的Android.mk。

例如下面的情形。

注意LOCAL_SRC_FILES。

如果是这样的，声明了src来源于src目录下的所有.java，那么你可以放心的使用上面的方法，但如果是

那说明src来源于项目目录下所有的.java文件，此时gen目录下的R.java也会被包含进去。那么在安卓源码环境下编译的时候会生成重复的R.java，导致R类重复的冲突。
这个时候需要根据情况修改Android.mk中的LOCAL_SRC_FILES。
具体怎么修改其实很简单，看项目结构就一目了然了。

通常上面的情形只会在导入APK项目时有很小的几率会出现，jar项目由于没有资源文件，一般没有问题，可以放心导入。
